# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in text2ontology, please report it responsibly.

**Do NOT** open a public GitHub issue for security-related concerns.

Instead, email: **security@text2ontology.com** (or open a private security advisory via GitHub Security tab)

Please include:
- Description of the vulnerability
- Steps to reproduce
- Affected component(s)
- Suggested fix (if you have one)
- Your contact info (for follow-up; we credit reporters in the changelog unless you ask otherwise)

## What to Expect

- Acknowledgement within 72 hours
- Initial assessment within 7 days
- Coordinated disclosure timeline depending on severity (typically 30-90 days before public disclosure)

## Supported Versions

text2ontology is in active development. Security fixes are applied to the latest `main` branch only at this stage. Once we reach a stable v1.0, we will publish a formal supported-versions matrix.

## Out of Scope

The following are not considered vulnerabilities:

- Issues in third-party dependencies (please report to upstream)
- Self-XSS that requires the victim to paste content into their own browser
- Missing security headers on documentation sites
- Theoretical issues without working exploits
- Best practice recommendations (open a regular issue for these)

## Hall of Fame

Reporters who responsibly disclose valid security issues will be credited here (with their permission).
